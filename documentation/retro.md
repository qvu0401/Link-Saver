# Project Restropective

## What went well

Learning FastAPI and writing the API endpoints was straightforward since the
data model and endpoint templates were already designed in design.md

Although the models and endpoints still required some tweaking, the general
outline made the coding much smoother and organized.

## What could be improved

The frontend (e.g. which features should call the endpoints, which
should be client-sided) was not well-designed beforehand so there were more
bug-fixing needed compared to the backend.

## What I would do differently next time

I realized how important system design is. Thoroughly planning the API endpoints, frontend elements, the URL lifecycle, and the overall user experience before starting to code can make the whole process much smoother. Next time I'll try throughly designing the major features of the app first, build that out, then handle any additional features afterward in separate sprints.
